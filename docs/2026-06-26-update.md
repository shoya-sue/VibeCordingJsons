## 品質分析サマリー

4 レンズ評価:

- **Applicability**: 2.1.193 の 15 項目中、テンプレ利用者が設定/参照し得るのは 3 件（`autoMode.classifyAllShell` 設定キー、`OTEL_LOG_ASSISTANT_RESPONSES` env、`CLAUDE_CODE_DISABLE_BG_SHELL_PRESSURE_REAP` env）。残りは UI 改善・バグ修正・内部挙動で config surface を持たず Low/Skip。
- **Cross-file Consistency**: env / autoMode の文書化は README の env テーブル + Other Settings テーブルに集約されている（`AGENTS.md` / `copilot-instructions.md` / `template/CLAUDE.md` には env テーブルが無い）。よって companion file 更新は不要、README 単独で一貫性が保てる。`autoMode.classifyAllShell` は既存の `autoMode.hard_deny` 行の隣が正位置。
- **Risk**: 3 件いずれも documentation-only・additive・opt-in。既存挙動を変えない（settings.json 非改変）。Low。ただし `OTEL_LOG_ASSISTANT_RESPONSES` には「未設定時 `OTEL_LOG_USER_PROMPTS` に追従 → prompt をログ済みの環境はアップグレードで応答内容も記録され始める」というアップグレード時の挙動変化があり、その注意を行内に明記する。
- **Completeness**: settings.json は autoMode を設定しないため `classifyAllShell` は「テンプレ設定値」ではなく「ドキュメント化された選択肢」として Other Settings に載せる（`hard_deny` と同じ扱い）。3 件とも README にのみ存在すればよく、テンプレ⇔install後の差異は生じない。

## 変更詳細

### #1 `autoMode.classifyAllShell`（README Other Settings）

`autoMode.hard_deny` 行の直後に追加:

```
| `autoMode.classifyAllShell` | auto モードで全 Bash/PowerShell コマンドを分類器に通す（既定は arbitrary-code-execution パターンのみ）。より厳格な auto モード運用向け（v2.1.193+） |
```

### #3 `OTEL_LOG_ASSISTANT_RESPONSES`（README env テーブル）

`OTEL_LOG_TOOL_DETAILS` 行の直後に追加（OTel log 系をグルーピング）:

```
| `OTEL_LOG_ASSISTANT_RESPONSES` | `claude_code.assistant_response` OTel log event にモデル応答テキストを含める（v2.1.193+）。**未設定時は `OTEL_LOG_USER_PROMPTS` に追従** → prompt をログ済みの環境はアップグレードで応答内容も記録され始める。prompts のみに保つなら `0` を明示 | `0`（prompts-only に保つ） |
```

### #6 `CLAUDE_CODE_DISABLE_BG_SHELL_PRESSURE_REAP`（README env テーブル）

`CLAUDE_CODE_RETRY_WATCHDOG` 行の直後（env テーブル末尾）に追加:

```
| `CLAUDE_CODE_DISABLE_BG_SHELL_PRESSURE_REAP` | メモリ逼迫時のアイドル background シェルコマンド自動回収を無効化（v2.1.193+、既定は自動回収有効） | `1`（自動回収を止める場合） |
```
