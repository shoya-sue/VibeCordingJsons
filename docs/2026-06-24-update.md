## 品質分析サマリー（4 レンズ）

### #1 `sandbox.credentials` setting（🟡 採用）
- **Applicability: High** — settings.json キーかつセキュリティ系。テンプレは既に sandbox を扱う（`settings.json` に `sandbox.failIfUnavailable`、README に `sandbox.allowAppleEvents` 行）。自然な companion。
- **Cross-file: README settings table のみ** — sandbox.* 系は README 表に集約されている（`allowAppleEvents` も README 表のみで CLAUDE.md には載せていない）。一貫性のため README 表に 1 行追加するに留める。
- **Risk: Low** — doc-only。`settings.json` は変更しない（`allowAppleEvents` と同様「テンプレは未設定＝無効」スタンス）。sandbox を能動有効化していないテンプレで default を変えるのは混乱を招くため。
- **Completeness** — README 表のみで完結。template/ と install 後の ~/.claude/ で差分なし。

### #6 remote MCP 5 分 abort + `CLAUDE_CODE_MCP_TOOL_IDLE_TIMEOUT`（🟡 採用）
- **Applicability: Medium** — 新規 env var。テンプレは MCP サーバーを 6 個同梱し remote サーバーも含むため、「5 分無応答で abort（従来は無限ハング）」という default 挙動変更 + override は実利あり。
- **Cross-file: README env table + CLAUDE.md Important Notes** — README env 表には既存の `MCP_TOOL_TIMEOUT` 行があり、その隣が最適配置。挙動変更そのものは CLAUDE.md Important Notes の MCP 系バレットに 1 行追加（`claude mcp login` の隣）。
- **Risk: Low** — CC ビルトインの改善（abort が default 化）であり additive。テンプレが制御するものではない。
- **Completeness** — env var 参照（README 表）＋挙動変更（CLAUDE.md）の 2 面で記載。

### ⚪ スキップ判断（主要なもの）
- **org-configured model restrictions**（model picker / `--model` / `/model` / `ANTHROPIC_MODEL`）— org-admin が managed settings で制御する runtime 挙動。テンプレ利用者が設定するキーではなく、既存の `enforceAvailableModels` / `availableModels` 行で概念カバー済み（Applicability: Low）。
- **`--json-schema` / workflow `agent({schema})` structured output 修正** — バグ修正。テンプレ設定なし。
- **CJK paste mojibake 修正** — 日本語関連だがバグ修正で設定項目なし。
- **subagent depth tracking 修正**（resumed が spawn depth 復元 / forked が cap にカウント）— 既存 5 階層 cap（`subagent-delegation.md` 記載）へのバグ修正。doc 変更不要。
