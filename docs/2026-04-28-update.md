## 変更詳細

### #1: PostToolUse `hookSpecificOutput.updatedToolOutput` 全ツール拡張（🟡）

**変更元 CC**: v2.1.121
**原因**: これまで `hookSpecificOutput.updatedToolOutput` で PostToolUse フックからツール出力を置換できるのは
MCP ツールのみだったが、v2.1.121 で全ツールに拡張された。hooks.md に追記する。

**Before**: hooks.md に `hookSpecificOutput.updatedToolOutput` の記述なし

**After**: hooks.md に以下セクションを追加:
```markdown
### PostToolUse ツール出力置換（v2.1.121+）

PostToolUse フックから `hookSpecificOutput.updatedToolOutput` を返すと、
全ツール（Bash, Read, MCP ツール等）のツール出力を置換できる。
```

---

### #2: Skills `${CLAUDE_EFFORT}` 変数（🟡）

**変更元 CC**: v2.1.120
**原因**: Skills の SKILL.md コンテンツ内で `${CLAUDE_EFFORT}` 変数を参照できるようになった。
現在の effort レベルに応じてスキルの動作を変えたい場合に有用。performance.md に追記する。

**Before**: performance.md の「スキル単位の effort 上書き」セクションに `${CLAUDE_EFFORT}` の記述なし

**After**: 以下の説明を追加:
```markdown
SKILL.md のコンテンツ内で `${CLAUDE_EFFORT}` 変数で現在の effort レベルを参照可能（v2.1.120+）。
```

---

### #3: `/ultrareview` CLI サブコマンド（🟡）

**変更元 CC**: v2.1.120
**原因**: `claude ultrareview [target]` サブコマンドが追加され、CI やスクリプトから
非インタラクティブに `/ultrareview` を実行できるようになった（`--json` でJSON出力、終了コード 0/1）。
template/CLAUDE.md の `/ultrareview` 説明文を更新する。

**Before**:
```
- `/ultrareview` — クラウドで並列マルチエージェント分析による包括的コードレビューを実行（引数なしで現ブランチ、`<PR#>` で特定 PR）
```

**After**:
```
- `/ultrareview` — クラウドで並列マルチエージェント分析による包括的コードレビューを実行（引数なしで現ブランチ、`<PR#>` で特定 PR）。CI からは `claude ultrareview [target]` サブコマンドで非インタラクティブ実行可能（`--json` でJSON出力）
```

---

### #4: `claude plugin prune` コマンド（🟡）

**変更元 CC**: v2.1.121
**原因**: `claude plugin prune` で孤立した自動インストール済みプラグイン依存を削除できるようになった。
`plugin uninstall --prune` でカスケード削除も可能。README のベストプラクティスに追記する。

**Before**: README に `claude plugin prune` の記述なし

**After**: README ベストプラクティスに追記

---

### #5: `alwaysLoad` MCP オプション（🟡）

**変更元 CC**: v2.1.121
**原因**: `.mcp.json` の MCP サーバー設定に `alwaysLoad: true` を指定すると、
そのサーバーの全ツールが tool-search の遅延対象から除外され、常に利用可能になる。
README の MCP/env 設定リファレンスに追記する。

**Before**: README に `alwaysLoad` オプションの記述なし

**After**: README の設定リファレンスに追記
