## 変更詳細

### #A `Tool(param:value)` 権限ルール構文（README.md）

**Before**（Supported patterns テーブル、`MCPSearch` 行で終わり）:

```
| `Skill(pattern)` | Skill execution | `Skill(explain-code:*)` |
| `MCPSearch` | MCP search | `MCPSearch` |
```

**After**（`Tool(param:value)` 行を追加 + テーブル後に注を追記）:

```
| `Skill(pattern)` | Skill execution | `Skill(explain-code:*)` |
| `Tool(param:value)` | Tool input parameter match | `Agent(model:opus)` |
| `MCPSearch` | MCP search | `MCPSearch` |
```

加えて glob ルール注の後に:

> **`Tool(param:value)` パラメータマッチ（v2.1.178+）**: ツール名に続けて `(param:value)` を書くと、そのツールの入力パラメータ値にマッチするルールを作れる（`value` 位置で `*` ワイルドカード可）。例: `Agent(model:opus)` を `deny` に置くと Opus を指定したサブエージェント起動をブロックできる。

### #C dynamic workflow キーワード起動挙動（template/CLAUDE.md）

**Before**: 「v2.1.160 で起動キーワードが `workflow` から `ultracode` に変更。「workflow」という語ではもう起動しない」

**After**: `ultracode` キーワードに加え、v2.1.178+ では "run a workflow" / "workflow:" のような明示句でも起動する（purple shimmer ハイライト、語の単なる言及では起動しない）旨に精度更新。

### #D auto mode 分類器（template/CLAUDE.md）

**Before**: auto mode 注は opt-in 同意とゲートウェイ向け env のみ。

**After**: v2.1.178+ で subagent spawn も起動前に分類器が評価し、ブロック対象アクションの要求を防ぐ旨を一節追記。
