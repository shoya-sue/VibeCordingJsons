# VibeCording Update — 2026-05-28

## 変更点サマリー

Claude Code **v2.1.153** (2026-05-28) を反映。本リリースは大半がバグ修正だが、唯一テンプレートに影響する user-facing 変更が **`/model` 挙動の再反転**。v2.1.144 で「`/model` はセッション限定、`d` キーで新セッションのデフォルト設定」となっていたが、v2.1.153 で IDE 挙動に揃えて反転し、**`/model` の選択はデフォルトで新セッションにも適用**、**現セッションのみは `s` キー**、keybinding `modelPicker:setAsDefault` → `modelPicker:thisSessionOnly` にリネーム（`d` アクションは `s` に置換）。これにより `template/CLAUDE.md` と `README.md` の既存注記が事実誤りになるため修正する。それ以外（plugin `skipLfs`、status line `COLUMNS`/`LINES`、各種バグ修正）はテンプレート非該当でスキップ。ECC バージョン変更なし（v1.10.0 据置）。

## 変更詳細

### #1 / #1b: `/model` デフォルト挙動の再反転訂正

v2.1.153 changelog（公式）:

> `/model` now saves your selection as the default for new sessions (matching the IDE). Press `s` in the picker to switch models for the current session only.
> If you customized the `modelPicker:setAsDefault` keybinding, rename it to `modelPicker:thisSessionOnly` in keybindings.json (the `d` action was replaced by `s`).

**`template/CLAUDE.md` line 45**

Before:
```
- `/model opusplan` — Auto-switch: Opus for planning, Sonnet for execution（v2.1.144+ で `/model` はセッション限定、ピッカーで `d` キーを押すと新セッションのデフォルトに設定）
```

After:
```
- `/model opusplan` — Auto-switch: Opus for planning, Sonnet for execution（v2.1.153+ で `/model` の選択はデフォルトで新セッションにも適用される。現セッションのみ切り替えたい場合はピッカーで `s` キー。旧 keybinding `modelPicker:setAsDefault` は `modelPicker:thisSessionOnly` にリネーム）
```

**`README.md` line 349**

Before:
```
> **v2.1.144+ 挙動変更**: `/model` で選んだモデルは現セッションのみ適用される。新セッションのデフォルトにしたい場合はモデルピッカーで `d` キーを押す。
```

After:
```
> **v2.1.153+ 挙動変更**: `/model` で選んだモデルはデフォルトで新セッションにも引き継がれる（IDE と同挙動）。現セッションのみ切り替えたい場合はモデルピッカーで `s` キーを押す。旧 keybinding `modelPicker:setAsDefault` は `modelPicker:thisSessionOnly` にリネーム（`d` アクションは `s` に置換）。
```
