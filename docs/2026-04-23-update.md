## 変更詳細

### #1: `mcp_tool` フックハンドラ追加（🟡）

**対象バージョン**: CC v2.1.118
**カテゴリ**: template/.claude/rules/ecc/common/hooks.md

CC v2.1.118 で新たなフックハンドラタイプ `mcp_tool` が追加された。
MCP ツールをフック内から直接呼び出せる形式で、`command` でシェルを立ち上げるより低コスト。

**Before:**
```markdown
| `command` | シェルコマンドを実行 |
| `http` | JSON を URL に POST（v2.1.69+） |
| `prompt` | Claude へのシングルターン評価（v2.1.85+） |
| `agent` | サブエージェントを起動してツール操作（v2.1.85+） |
```

**After:**
```markdown
| `command` | シェルコマンドを実行 |
| `http` | JSON を URL に POST（v2.1.69+） |
| `prompt` | Claude へのシングルターン評価（v2.1.85+） |
| `agent` | サブエージェントを起動してツール操作（v2.1.85+） |
| `mcp_tool` | MCP ツールを直接呼び出し（v2.1.118+） |
```

**対象バージョン**: CC v2.1.118


---

### #4: `DISABLE_UPDATES`（⚪）

**対象バージョン**: CC v2.1.118
**カテゴリ**: env var

`DISABLE_UPDATES=1` を設定すると CC の自動更新を無効化する。
CI/CD や固定バージョン運用向け。標準テンプレートでは不要。

### #5: `wslInheritsWindowsSettings`（⚪）

WSL 環境で Windows 側のマネージド設定を継承するフラグ。macOS 環境には関係なし。

### #6: `autoMode.$defaults`（⚪）

`autoMode.allow` / `soft_deny` / `environment` に `"$defaults"` を指定することで組み込みリストを拡張できる。
テンプレートの `settings.json` には `allowedDomains` が設定されていないため no-op。

### #7-9: CC 組み込み改善（⚪）

- Vim visual mode (`v`, `V`) — CC 組み込み UX
- カスタムテーマ（`/theme`、JSON ファイル） — 個人設定、デフォルト不要
- MCP OAuth `expires_in` なし問題修正 — CC 組み込みバグフィックス

### #10: ECC 新バージョンなし（⚪）

v1.10.0（2026-04-05 リリース）が最新。前回確認から変化なし。
