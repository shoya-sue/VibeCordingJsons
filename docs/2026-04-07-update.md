## 1. Claude Code の主要アップデート

### 1-1. フックシステムの大幅拡張

**As-Is（現在の full/settings.json）**

`full/.claude/settings.json` には 21 イベントが設定済み。
`full/.claude/rules/ecc/common/hooks.md` のドキュメントには以下の 3 種類のみ記載：

```
- PreToolUse: ツール実行前
- PostToolUse: ツール実行後
- Stop: セッション終了時
```

**To-Be（取り込むべき変更）**

実際のイベント数は **27 イベント**に増加（v2.1.84〜v2.1.92 で追加）。
新規追加イベント：

| 新規イベント | 追加バージョン | 説明 |
|-------------|--------------|------|
| `TaskCreated` | v2.1.84 | タスク作成時（ブロッキング） |
| `StopFailure` | v2.1.78 | Stop フックの失敗時 |
| `CwdChanged` | v2.1.83 | カレントディレクトリ変更時 |
| `FileChanged` | v2.1.83 | ファイル変更検知時 |
| `PermissionDenied` | v2.1.89 | 権限拒否時（`retry: true` 返却可） |

フック定義に新しいフィールドも追加：

```jsonc
// v2.1.85 追加: 条件付きフックフィルタリング
{
  "matcher": "Bash",
  "if": "Bash(git *)",          // ← NEW: 権限ルール構文でフィルタ
  "hooks": [...]
}

// v2.1.85 追加: PreToolUse でインプット変更
{
  "type": "command",
  "command": "...",
  "returnPolicy": "updatedInput" // ← NEW: permissionDecision と組み合わせ
}
```

新しいフックハンドラータイプ（`command` 以外）：

| タイプ | 説明 |
|--------|------|
| `http` | JSON を URL に POST |
| `prompt` | Claude へのシングルターン評価 |
| `agent` | サブエージェントを起動してツール操作 |

### 4-3. カスタムエージェント（`.github/agents/`）の仕様変更

**As-Is（VibeCording full パターン）**

`.github/agents/` の `.agent.md` ファイルに `infer` フィールドを使用している可能性。
`tools` フィールドに `grep`, `glob`, `view`, `bash` などのツール名を直接記載。

**To-Be**

| 変更点 | 内容 |
|--------|------|
| `infer` フィールド | **廃止** → `disable-model-invocation: false` + `user-invocable: true` に移行 |
| `tools` エイリアス | `grep`/`glob`/`view`/`bash` は引き続き動作するが、正式エイリアスは `read`/`search`/`execute` |
| `model` フィールド | 任意指定が可能（例: `claude-sonnet-4-6`） |

最新のエージェントフロントマター例：

```yaml
---
name: "Code Reviewer"
description: 'レビューエージェント'
tools: ["read", "search", "execute"]
user-invocable: true
disable-model-invocation: false
---
```

### 4-4. インストラクションシステムの階層化（新機能）

**As-Is**

`.github/copilot-instructions.md` 1 ファイルのみ（常時適用）。

**To-Be**

3 層構造に拡張された：

| 場所 | スコープ | 適用条件 |
|------|---------|---------|
| `.github/copilot-instructions.md` | リポジトリ全体 | 常時 |
| `.github/instructions/*.instructions.md` | パス限定 | `applyTo` glob にマッチしたとき |
| `.github/skills/*.md` | オンデマンド | エージェントが必要と判断したとき |

`.instructions.md` のフロントマター：

```yaml
---
description: 'Ruby モデルのコーディング規約'
applyTo: "app/models/**/*.rb"      # パスターゲティング
excludeAgent: "code-review"        # 特定エージェントから除外
---
```

**組織レベルのインストラクション（GA: 2026-04-02）**

GitHub Business/Enterprise 管理者が組織全体のデフォルトインストラクションを設定可能になった。

### 4-5. VibeCording テンプレートへの影響

**As-Is / To-Be 比較**

| 対象 | As-Is | To-Be |
|------|-------|-------|
| `copilot-instructions.md` の `gh copilot` コマンド記述 | `gh copilot suggest` / `explain` | 削除（コマンド廃止）|
| エージェントの `infer` フィールド | 使用している場合がある | `user-invocable: true` + `disable-model-invocation: false` に置換 |
| `tools` フィールド | `grep`, `glob`, `view`, `bash` | `read`, `search`, `execute`（後方互換あり） |
| `full/` パターンのディレクトリ構造 | `.github/agents/` のみ | `.github/instructions/`, `.github/skills/` の追加を検討 |
| インストール手順（README等） | `gh extension install github/gh-copilot` | `gh copilot` または `brew install github-copilot-cli` |
