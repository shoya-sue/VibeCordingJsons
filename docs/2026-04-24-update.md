## 変更詳細

### #1: PostToolUse `duration_ms` 入力追記（🟡）

**対象バージョン**: CC v2.1.119
**カテゴリ**: template/.claude/rules/ecc/common/hooks.md

CC v2.1.119 で PostToolUse/PostToolUseFailure フックの入力 JSON に `duration_ms` フィールドが追加された。
ツール実行時間（権限確認や PreToolUse フックの時間は除く）をミリ秒で受け取れる。

パフォーマンス監視・遅延ログ・メトリクス収集系の command フックでそのまま活用できるため、
`Advanced Features` セクションに短く追記する。

**After（追加セクション）:**
```markdown
### PostToolUse `duration_ms` 入力 (v2.1.119+)

PostToolUse/PostToolUseFailure フックの入力 JSON に `duration_ms`（ツール実行時間、ms）が含まれる。
権限確認と PreToolUse フック実行時間は除外される。パフォーマンス監視系フックで利用可能。
```

---

### #2: `prUrlTemplate`（⚪）

GitHub Enterprise、GitLab、Bitbucket 等の非 github.com プロバイダで PR フッタバッジ URL をカスタマイズする設定。
標準テンプレートは公開 github.com 想定のため追加不要。

### #3: `CLAUDE_CODE_HIDE_CWD`（⚪）

起動ロゴから作業ディレクトリパスを隠すプライバシー向けオプション。
デフォルトの表示動作で問題ないため標準テンプレートに設定不要。

### #4: `ENABLE_TOOL_SEARCH`（⚪）

Vertex AI 経由利用時、デフォルト無効化された Tool Search をオプトインで有効化。
Anthropic 直接 API を使う標準テンプレートには無関係。

### #5-11: CC 組み込み改善・バグ修正（⚪）

以下はすべて CC 自体のバージョンアップで自動適用されるため、テンプレート設定への変更不要:

- `/config` の settings.json 永続化（プロジェクト/ローカル/ポリシー優先順）
- `--from-pr` の GitLab/Bitbucket/GHE URL 対応
- `/skills` Enter 動作修正、`/plan` 既存プラン使用、`/reload-plugins` 無効プラグイン警告抑制、`/export` 実モデル表示
- `--print` モードでエージェントの `tools:`/`disallowedTools:` frontmatter を尊重
- `--agent <name>` で built-in エージェントの `permissionMode` を尊重
- Agent tool の `isolation: "worktree"` が古い worktree を再利用する問題修正
- auto-compaction 前に起動したスキルが次メッセージで再実行される問題修正
- MCP `${ENV_VAR}` ヘッダ置換、OAuth `client_secret_post` 交換、plugin MCP optional `user_config` 対応
- Subagent/SDK MCP の並列再構成、無効 MCP の `/status` 誤表示修正
- PowerShell ツール Bash パリティ（Windows）
- OpenTelemetry `tool_use_id` / `tool_input_size_bytes`
- ステータスライン stdin `effort.level` / `thinking.enabled`
- `blockedMarketplaces` パターン修正
