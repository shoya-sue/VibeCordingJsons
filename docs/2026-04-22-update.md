## 変更詳細

### #1: `CLAUDE_CODE_FORK_SUBAGENT=1`（⚪）

**対象バージョン**: CC v2.1.117
**カテゴリ**: env var

外部ビルド環境でのフォーク型サブエージェントを有効化するオプション env var。
CI/CD パイプライン等の特殊な外部ビルド環境向けであり、通常の Claude Code 利用には不要。
テンプレートのデフォルト設定に追加する価値なし。

### #2: effortLevel デフォルト変更（⚪）

**対象バージョン**: CC v2.1.117
**カテゴリ**: settings.json

Pro/Max プランにおいて Opus 4.6 / Sonnet 4.6 のデフォルト effort が `medium` → `high` に変更された。
テンプレートの `template/.claude/settings.json` は既に `"effortLevel": "high"` を明示設定しているため no-op。

### #3: Opus 4.7 コンテキストウィンドウ修正（⚪）

**対象バージョン**: CC v2.1.117
**カテゴリ**: CC バグフィックス

Opus 4.7 のコンテキスト計算が誤って 200K を使っていた問題を修正し、ネイティブ 1M ウィンドウを使用するよう変更。
autocompact のタイミングが正しくなる副次効果あり。CC 組み込み修正のため設定変更不要。

### #4-6: その他 CC v2.1.117 改善（⚪）

**カテゴリ**: CC 組み込み改善・バグフィックス

- OAuth 401 時のトークンリフレッシュ（セッション断絶防止）
- MCP 並列起動デフォルト化（起動高速化）
- WebFetch 大規模 HTML ハング修正
- NO_PROXY 設定の Bun 経由リクエストへの適用
- Linux idle 再レンダリングループのメモリリーク修正
- プロキシ HTTP 204 エラーハンドリング改善
- Bedrock + Opus 4.7 + thinking-disabled の 400 エラー修正

いずれも CC 組み込み変更のため、テンプレート設定ファイルへの変更は不要。
