# VibeCording Update — 2026-05-11

## 変更点サマリー

obsidian-mcp の起動信頼性を大幅改善。`npx -y` 経由から `npm install -g` グローバルバイナリに切り替え、install.sh がバイナリを自動検出して絶対パスに置換するよう拡張。プロジェクト固有の settings.json を追加し settings.local.json を整理。

## 主な変更

### obsidian-mcp: npx → グローバルバイナリ

**背景**: `npx -y obsidian-mcp` は毎起動時に npm レジストリへの疎通確認が発生し、サブプロセス env scrubbing（`CLAUDE_CODE_SUBPROCESS_ENV_SCRUB`）環境ではタイムアウトしやすかった。

**対応**:
- `template/.mcp.json`: `command` を `obsidian-mcp`（グローバルバイナリ名）に変更、`PATH` に `NODEJS_BIN_DIR` プレースホルダーを追加
- `install.sh`: obsidian-mcp グローバルインストール検出ブロックを追加
  - 既存バイナリがあればそのまま使用
  - なければ `npm install -g obsidian-mcp` を自動実行
  - バイナリパスと Node.js bin ディレクトリを `.mcp.json` に展開

### playwright: -y フラグ欠落修正

`.mcp.json` の playwright npx コマンドに `-y` フラグが抜けていたため、初回起動時にインタラクティブプロンプトが出る可能性があった。修正済み。

### .claude/settings.json 追加（プロジェクト固有）

- `Bash(./install.sh *)` — インストーラー実行を自動承認
- `Bash(claude mcp *)` — MCP 管理コマンドを自動承認
- `Bash(claude --print-config)` — 設定確認コマンドを自動承認
- `WebSearch` — ウェブ検索を自動承認

### settings.local.json 整理

デバッグセッションで蓄積した一時的なパターン（python3 デバッグ、xargs 系、一回限りの MCP テスト）をすべて除去。グローバル `~/.claude/settings.json` と重複しているエントリも削除。

### .gitignore: .claude/worktrees/ を除外

Claude Code がエージェント隔離のために生成するワークツリーをリポジトリから除外。

### npx MCP サーバー: `${PATH}` → 絶対パス指定

**背景**: `.mcp.json` の `env.PATH` に `${PATH}` を設定していたが、`CLAUDE_CODE_SUBPROCESS_ENV_SCRUB` 環境下では変数展開されず、npx のパスが通らないため context7・playwright・excalidraw・github の各 MCP サーバーが起動失敗していた。

**対応**:
- `template/.mcp.json`: 全 npx サーバーの `env.PATH` を `NODEJS_BIN_DIR:/opt/homebrew/bin:/usr/local/bin:/usr/bin:/bin` に変更。`NODEJS_BIN_DIR` は install.sh が obsidian-mcp 検出時に実際の Node.js bin ディレクトリに展開済み。
- `.mcp.json`（プロジェクト用）・`~/.mcp.json`（グローバル）: NVM 絶対パス `/Users/shoya-sue/.nvm/versions/node/v24.9.0/bin` に直接置換。
- **根本原因**: `${PATH}` は MCP env フィールドでは展開されない。env scrubbing の有無に関わらず、npx を呼ぶサーバーには常に絶対パスを含む `PATH` を明示する必要がある。
