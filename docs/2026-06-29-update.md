## #2 検証: 2.1.195 は既存テンプレ matcher を壊さないか（破壊的変更チェック）

2.1.195 の matcher 変更は「**ハイフン付き識別子**が部分一致→完全一致」。テンプレが出荷する全 hook matcher を監査:

| settings.json hook | matcher | 種別 | 2.1.195 影響 |
|---|---|---|---|
| PreToolUse | `Bash` | exact（ハイフン無し） | なし |
| PostToolUse | `Write\|Edit` | pipe list（ハイフン無し） | なし |
| PostToolUseFailure | `mcp__obsidian__*` → `mcp__obsidian__.*` | 正規表現（サーバー名 `obsidian` ハイフン無し） | なし |

→ **テンプレは 2.1.195 で壊れない**（唯一の MCP hook matcher がハイフン無しサーバー `obsidian`、exact-match 化の対象外）。`permissions.allow` の `mcp__github__*` 等はハイフン無し&別サブシステム（権限ルール glob、hook regex とは別エンジン）で無関係。

ただし公式 hooks docs によれば、サーバー全ツールにマッチさせる正規形は **`mcp__obsidian__.*`**（regex `.*`）。テンプレの `mcp__obsidian__*` は regex `mcp__obsidian_` + `_*` と解釈され**偶然動いていた**（壊れてはいない）が、今回 hooks.md に追記した「全ツールマッチは `mcp__brave-search__.*` を使う」という指針と出荷 config が不一致。**Lens 2/Lens 4 の取りこぼし**として #A で正規形に修正。CRITICAL/破壊的ではなく MEDIUM 級（同じ obsidian ツール群にマッチ、より厳密になるだけ・リスクほぼゼロ）。
