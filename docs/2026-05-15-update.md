## 🔥 追補 (2026-05-15 夕方): Obsidian MCP を撤去 → cyanheads で再採用

### 経緯

1. **朝の段階での状態**: 2026-05-13 に `StevenStavrakis/obsidian-mcp v1.0.6` を撤去済み (Write/Edit + claude-obsidian プラグイン体制)
2. **疑問の発端**: 「Obsidian アプリの Settings → Command line interface を ON にすれば直るのでは?」というユーザー仮説
3. **徹底調査**: 仮説を否定 (Issue #36 未実装、Obsidian CLI と obsidian-mcp は別物)。真の原因は v1.0.6 の既知バグ 2 件:
   - Issue #37 — ConnectionMonitor が 70 秒無操作で silent close
   - Issue #48 — `edit-note` の discriminated union が `zodToJsonSchema` で空 properties として送信される
4. **「Anthropic 公式 Obsidian MCP」を探索 → 不在を確認**: 公式 reference servers は `filesystem/git/memory/time` 等のジェネリック 7 種のみで Obsidian 用は存在しない
5. **Obsidian Community Plugin Store 登録の MCP プラグイン 8 個を比較**:
   - 🥇 **REST and MCP server** (`dsebastien/obsidian-cli-rest`) — Plugin Store 登録、Plugin 単体で完結
   - 🥈 **Local REST API + cyanheads/obsidian-mcp-server** — Local REST API が事実上の標準 + cyanheads が 14 ツール完備
   - 他 6 個 (Nexus, MCP Vault Bridge, mcp-rest, Hydrate, MiYo Kado, HangarX, Fast Note Sync)
6. **採用決定**: cyanheads + Local REST API (機能・surgical edit 強さで上)
