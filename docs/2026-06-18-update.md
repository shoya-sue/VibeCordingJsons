# VibeCording Update — 2026-06-18

## 変更点サマリー

Claude Code **2.1.181** 対応。2.1.180 は **欠番（未リリース）**で、カバレッジは 2.1.179（v0.62.0 でカバー済み）から 2.1.181 へ直接進む。2.1.181 は大半が runtime バグ修正・UX 改善だが、テンプレート（設定リポジトリ）に文書化価値のある **新規 capability 3 件**を含む:

1. `/config key=value` — プロンプトから任意の設定を直接変更する slash command 構文
2. `CLAUDE_CLIENT_PRESENCE_FILE` — モバイル通知を抑制する新規 environment variable
3. `sandbox.allowAppleEvents` — macOS の sandboxed command で Apple Events を許可する opt-in settings key

いずれも **doc-only / additive / 低リスク**。`settings.json` の値・挙動は無変更（sandbox は引き続き `failIfUnavailable` のみで `allowAppleEvents` は文書化のみ）。ECC は **2.0.0** 据え置き（ローカル＝テンプレ＝2.0.0、変更なし）。

## 品質分析サマリー

2.1.181 の全エントリを 4 レンズで評価。

- **Lens 1（適合性）**: 大半（Bun 1.4 / streaming 表示 / auto-retry / subagent パネル UX / MCP OAuth ページ / fullscreen URL クリック / 約 30 件の Fixed）は runtime/UX で、ユーザーが設定・記述する capability ではない → **Skip**。文書化価値があるのは新規 slash command 構文 1 件、新規 env var 1 件、新規 settings key 1 件の計 **3 件**。
- **Lens 2（横断一貫性）**: `/config key=value` は CLAUDE.md「Slash Commands」が canonical（README に slash command 表は無い）。`CLAUDE_CLIENT_PRESENCE_FILE` は README「env (Environment Variables)」表が canonical。`sandbox.allowAppleEvents` は README「Other Settings」表が canonical（niche/opt-in/managed キーを集約する既存の場所）。CLAUDE.md の Important Notes は env/settings の全件ミラーではない（高インパクト項目のみ）ため、3 件いずれも CLAUDE.md 側の companion 追記は不要。
- **Lens 3（導入リスク）**: 3 件すべて additive な文書追記で既存挙動を変えない（Low）。特に `allowAppleEvents` は **settings.json に追加せず文書化のみ**とし、macOS ユーザーの sandbox 挙動を変えない（テンプレの sandbox 最小主義＝`denyRead`/`allowRead` 不採用方針と整合）。
- **Lens 4（適用完全性）**: `/config` は CLAUDE.md で `/workflows` 行から「Workflow keyword trigger」として言及済みだが、独立した `/config` エントリは未存在 → 新規追加。`CLAUDE_CLIENT_PRESENCE_FILE` / `allowAppleEvents` は README 双表に未記載 → 追加。`settings.json` の `sandbox` ブロックは変更しない（意図的）。

結論: **🟡 推奨 3 件を適用**（CLAUDE.md ×1、README ×2）。🔴 必須・settings.json 変更・ECC 変更はなし。
