## Opus 4.8 移行に伴う設定・hooks 全監査（追補）

ユーザー要請により settings.json の全 env / 全設定キー / 全 hooks を Opus 4.8 観点で監査。

**結論: 削除が必要な設定・hook は無い。**
- `CLAUDE_CODE_OPUS_4_6_FAST_MODE_OVERRIDE`（非推奨・06-01 削除予定）は **元々 settings.json に未設定**。ドキュメント（README/performance.md）のみ記載 → 非推奨明記で対応済み。
- hooks 9 イベントはすべて ECC スクリプト/ローカルスクリプトで、廃止 env を参照しない。`obsidian-auto-capture.sh` の `--model claude-haiku-4-5` ハードコードは Opus 4.8 と無関係で継続 OK。
- `model: "opus"` エイリアスが自動で Opus 4.8 に解決（昇格の本体）。`effortLevel: "xhigh"` は Opus 4.8 で健在。

**精度修正 2 件（本 PR に追加）:**
- `CLAUDE_CODE_AUTO_COMPACT_WINDOW=1000000`: `gh issue view 43989` で確認 → **issue は今も OPEN（未修正）**、正式タイトルは "v2.1.92 regression: autocompact threshold reduced to 400k on **Opus 4.6** (1M context)"。workaround は維持が正しい。README コメントを「Opus 4.x」決め打ちから「1M context の Opus モデル全般・未修正」へ一般化。
- `CLAUDE_CODE_MAX_OUTPUT_TOKENS=64000`: Opus 4.8 は API で最大 **128k** 出力対応（4.7 までは 64k 系）。**設定値は 64000 維持**（ユーザー判断: 保守的・予測可能）。README/CLAUDE.md に「長文出力が多ければ settings.local.json で 128000 まで引き上げ可、Claude Code 内部 cap 時は無効」と注記。
