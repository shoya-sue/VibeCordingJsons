## 変更詳細

### #1: `sandbox.network.deniedDomains`（⚪）

**対象バージョン**: CC v2.1.113
**カテゴリ**: settings.json

特定ドメインへのネットワークアクセスをブロックする新設定キー。`allowedDomains` ワイルドカードが設定されている場合でも特定ドメインを除外できる。

テンプレートは `allowedDomains` を設定していないため、`deniedDomains` を追加しても効果がない。ユーザーが `allowedDomains` を設定した場合に `sandbox.network.deniedDomains` で特定ドメインを除外できることをコメントで示す価値はあるが、デフォルト動作に影響しないため追加不要と判断。

```json
// 追加可能なフォーマット（必要な場合のみ）
"sandbox": {
  "failIfUnavailable": false,
  "network": {
    "deniedDomains": ["malicious.example.com"]
  }
}
```

### #2: ECC v1.10.0 スキル/エージェント数更新（⚪）

**対象バージョン**: ECC v1.10.0（2026-04-05 リリース）
**カテゴリ**: README, CLAUDE.md, settings.json

ECC v1.10.0 での変更:
- スキル: 136 → 156（新規: brand-voice, social-graph-ranker, connections-optimizer, manim-video, remotion-video-creation, nestjs-patterns）
- エージェント: 30 → 38（operator ワークフロー lanes 追加）
- ECC 2.0 alpha control-plane (`ecc2/`) 追加
- SQLite state store 追加
- Windows CI 完全対応

**ローカル未インストールのため全変更をスキップ。** install.sh が `~/.claude/plugins/cache/everything-claude-code/everything-claude-code/` から最新バージョンを auto-detect するため、ECC 1.10.0 をインストールすれば自動的に適用される。

### #3: CC v2.1.113 Bash ルール改善（⚪）

**対象バージョン**: CC v2.1.113
**カテゴリ**: CC 組み込み動作

- `env`/`sudo`/`watch`/`ionice`/`setsid` でラップされたコマンドに deny ルールが適用されるように
- macOS: `/private/{etc,var,tmp,home}` パスが `Bash(rm:*)` ルールで危険ターゲットとして扱われる
- `Bash(find:*)` ルールが `find -exec`/`-delete` を自動承認しなくなった（より安全に）

テンプレート設定変更不要。CC v2.1.113 以降で自動的に適用。

CC 組み込み機能。テンプレート変更不要。
