## 変更詳細

### #1 — `CLAUDE_CODE_NO_FLICKER=1` ドキュメント追加 (v2.1.91)

チラつきなしの alt-screen レンダリングを有効化する環境変数。`performance.md` の拡張思考セクションに追記。

**Before:** 記載なし

**After:**
```markdown
- **No-flicker mode**: `CLAUDE_CODE_NO_FLICKER=1` — チラつきなし alt-screen レンダリング（v2.1.91+）
```

---

### #2 — `effort` スキル・フロントマター (v2.1.80)

スキル/コマンドの SKILL.md フロントマターに `effort: low|medium|high` を指定すると、そのスキル実行時だけ思考レベルを上書きできる。

**Before:** 記載なし

**After:**
```markdown
### スキル単位の effort 上書き

SKILL.md の frontmatter で個別スキルの effort を指定可能（v2.1.80+）:
```yaml
effort: low   # このスキル実行時は low に固定
```
軽量なユーティリティ系スキルに `low` を指定するとコスト削減になる。
```

---

### #3 — `WorktreeCreate` HTTP レスポンス対応 (v2.1.84)

`WorktreeCreate` フックが `type: "http"` に対応し、`hookSpecificOutput.worktreePath` を返せるようになった。

**Before:**
```
| `WorktreeCreate` | ワークツリー作成時 | No |
```

**After:**
```
| `WorktreeCreate` | ワークツリー作成時（`type:"http"` で `worktreePath` 返却可） | No |
```
