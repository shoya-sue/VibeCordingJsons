## 変更詳細

### #1: `/tui` コマンド追加（🟡 推奨）

**対象バージョン**: CC v2.1.110
**カテゴリ**: CLAUDE.md / Slash Command

`/tui fullscreen` でチラつきなしの全画面レンダリングに切り替え。現在の会話を維持したまま起動可能。

**Before:**
```
- `/recap` — 離席後のセッションサマリーを手動表示
```

**After:**
```
- `/tui [fullscreen]` — チラつきなし全画面レンダリングに切り替え
- `/focus` — フォーカスビュー表示切り替え（Ctrl+O はノーマル/詳細トランスクリプト切り替えのみ）
- `/recap` — 離席後のセッションサマリーを手動表示
```

---

### #2: `/focus` コマンド追加（🟡 推奨）

**対象バージョン**: CC v2.1.110
**カテゴリ**: CLAUDE.md / Slash Command

v2.1.110 で `Ctrl+O` がノーマル/詳細トランスクリプト切り替えのみになり、フォーカスビューは `/focus` コマンドで操作するように変更。#1 と同時適用。

---

### #3: `autoScrollEnabled` 追加（🟡 推奨）

**対象バージョン**: CC v2.1.110
**カテゴリ**: settings.json キー

全画面モードで会話の自動スクロールを制御。デフォルト `true`。明示化することでカスタマイズ可能性を示す。

**Before:**
```json
"disableSkillShellExecution": false,
```

**After:**
```json
"autoScrollEnabled": true,
"disableSkillShellExecution": false,
```

---

### #7: `effortLevel xhigh` ドキュメント更新（🟡 推奨）

**対象バージョン**: CC v2.1.111
**カテゴリ**: performance.md / settings.json キー記述

performance.md の `effortLevel` 列挙に `"xhigh"` を追加（`high` と `max` の間）。

**Before:**
```
- **Config**: Set `effortLevel: "low" | "medium" | "high"` in settings.json (v2.1.68+, default restored to `"high"` in v2.1.94)
```

**After:**
```
- **Config**: Set `effortLevel: "low" | "medium" | "high" | "xhigh"` in settings.json (v2.1.68+, `"xhigh"` added in v2.1.111 for Opus 4.7, default `"high"`)
```
